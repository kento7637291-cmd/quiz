# 負荷テスト ガイド

100人の同時ユーザーでサーバーのパフォーマンスをテストします。

## 📋 テスト環境
- **メモリ制限**: 512 MB
- **同時ユーザー数**: 100人
- **テスト時間**: 60秒（カスタマイズ可能）
- **主なシナリオ**: ランキング表示、コメント投稿・取得

## ⚙️ カスタマイズ方法

### Python版スクリプトを編集

**`load_test_python.py`** の冒頭を編集：

```python
# テスト設定
HOST = "http://localhost:8080"           # サーバーアドレス
NUM_USERS = 100                          # ユーザー数（変更可）
TEST_DURATION = 60                       # テスト時間 秒（変更可）
CONCURRENT_REQUESTS = 10                 # 同時実行数（変更可）
```

### 例：50人で120秒テスト
```python
NUM_USERS = 50
TEST_DURATION = 120
```

## 🎯 テスト結果の解釈

### 成功の目安
| 指標 | 良好 | 要注意 | 問題 |
|------|------|--------|------|
| 失敗率 | < 5% | 5-10% | > 10% |
| 平均応答時間 | < 200ms | 200-500ms | > 500ms |
| 95%ile | < 500ms | 500-1000ms | > 1000ms |
| メモリ使用量 | < 256MB | 256-400MB | > 512MB |

### 対応策

#### 失敗率が高い場合
- データベース接続プーリングを確認
- Socket.io イベントハンドラのボトルネック確認
- ログを確認（`instance/app.log` など）

#### メモリ使用量が多い場合
- セッションストレージの最適化
- 不要なデータキャッシュの削除
- ガベージコレクション設定確認

#### 応答時間が遅い場合
- SQL クエリの最適化
- キャッシング機構の導入
- 静的ファイル配信の最適化（CDN等）

## 🔧 トラブルシューティング

### エラー: Connection refused
```
Traceback: requests.ConnectionError: HTTPConnectionPool(host='localhost', port=8080)
```
**対応**: アプリが起動しているか確認
```bash
curl http://localhost:8080/
```

### エラー: ModuleNotFoundError: No module named 'requests'
```bash
pip3 install requests
```

### テスト中に HTTP 500 エラーが多発
- サーバーログを確認
- データベース接続限界に達している可能性
